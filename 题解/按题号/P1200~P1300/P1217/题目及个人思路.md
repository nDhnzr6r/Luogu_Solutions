# P1217 [USACO1.5] 回文质数 Prime Palindromes

## 题目描述

因为 $151$ 既是一个质数又是一个回文数（从左到右和从右到左是看一样的），所以 $151$ 是回文质数。

写一个程序来找出范围 $[a,b] (5 \le a < b \le 100,000,000)$（一亿）间的所有回文质数。

## 输入格式

第一行输入两个正整数 $a$ 和 $b$。

## 输出格式

输出一个回文质数的列表，一行一个。

## 输入输出样例 #1

### 输入 #1

```
5 500

```

### 输出 #1

```
5
7
11
101
131
151
181
191
313
353
373
383

```

Hint 1: Generate the palindromes and see if they are prime.

# 个人做题思路

## 核心思路

先构造回文数，再判断质数。因为范围内的回文数远少于质数。

## 思路演变

我最开始的思路是枚举所有奇数，判断是否是回文数再去判断质数，但一直在超时。于是我疯狂优化判断质数的代码，使用了 6k±1 判断法，但最后一个测试点还是超时。

然后我转变思路：与其枚举所有数筛回文，不如直接构造回文数再去判断质数。这样复杂度从 O(b) 降到 O(√b)，成功通过。

## 具体步骤

1. 读入 a、b
2. 枚举 k 从 1 到 9998，用字符串拼接法构造奇长度回文数
   - `s = to_string(k)`，`ss = reverse(s).erase(0,1)`
   - 拼接 `s + ss` 得到回文数
3. 判断回文数是否在 [a,b] 范围内
4. 在范围内则用 6k±1 法判断质数：跳过 2、3 的倍数，从 j=5 开始每步+6 检查 j 和 j+2
5. 是质数则输出
6. 11 是唯一的偶长度回文质数，特殊处理（接在 7 后面输出）

## 踩坑点

- 枚举法中 $10^8$ 范围太大，即使优化质数判断也会超时
- 回文数构造只生成奇长度即可，所有偶长度回文数都能被 11 整除（11 本身除外）
- 质数判断用 `1LL * j * j <= x` 防止 int 溢出
- 跳过 2、3 的倍数后再用 6k±1 能减少判断次数
