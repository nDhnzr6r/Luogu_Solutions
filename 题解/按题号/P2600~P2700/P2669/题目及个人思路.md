# P2669 [NOIP 2015 普及组] 金币

## 题目背景

NOIP2015 普及组 T1

## 题目描述

国王将金币作为工资，发放给忠诚的骑士。第一天，骑士收到一枚金币；之后两天（第二天和第三天），每天收到两枚金币；之后三天（第四、五、六天），每天收到三枚金币；之后四天（第七、八、九、十天），每天收到四枚金币……；这种工资发放模式会一直这样延续下去：当连续 $n$ 天每天收到 $n$ 枚金币后，骑士会在之后的连续 $n+1$ 天里，每天收到 $n+1$ 枚金币。

请计算在前 $k$ 天里，骑士一共获得了多少金币。

## 输入格式

一个正整数 $k$，表示发放金币的天数。

## 输出格式

一个正整数，即骑士收到的金币数。

## 输入输出样例 #1

### 输入 #1

```
6
```

### 输出 #1

```
14
```

## 输入输出样例 #2

### 输入 #2

```
1000

```

### 输出 #2

```
29820

```

## 说明/提示

**【样例 1 说明】**

骑士第一天收到一枚金币；第二天和第三天，每天收到两枚金币；第四、五、六天，每天收到三枚金币。因此一共收到 $1+2+2+3+3+3=14$ 枚金币。


对于 $100\%$ 的数据，$1\le k\le 10^4$。

---

# 个人做题思路

## 核心思路

模拟金币发放规律：每轮连续 i 天发 i 个金币，累计天数直到第 k 天。

## 具体步骤

1. 读入 k
2. 外层循环 i 表示当前每天发 i 个金币
3. 内层循环连续 i 天，每天累加 i 个金币
4. 计数器 n 记录已发天数，当 `n == k` 时输出 `s` 并结束

## 踩坑点

- 双重循环中内层 `j <= i` 控制连续天数
- 达到 k 天后直接 `return 0` 退出，避免多余循环
- k 最大 $10^4$，总金币数不会爆 int
